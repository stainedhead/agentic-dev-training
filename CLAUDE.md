## What This Project Is

A 12-module enterprise developer training programme on agentic AI practices. Each module is a 12-slide `.pptx` deck built by a Node.js script using `pptxgenjs`. The slides are professional, consistent, and regenerable from source.

**Two kinds of content:**
- `modules/*.md` — human-readable reference guides (one per module, editable directly)
- `slides/*.pptx` — presentation decks (generated by `build/module*.js` scripts)

## Build System

### Install

```bash
cd build && npm install
```

### Preferred: use Make (incremental)

```bash
make              # build only slides whose MD is newer than the PPTX
make clean        # remove all slides/*.pptx and build/*.pptx
make clean-all    # clean + remove npm sentinel (forces reinstall on next build)
make install      # run npm install explicitly
```

Make compares timestamps automatically — if the PPTX is newer than the MD, that module is skipped. Run from the repo root (not inside `build/`).

### Build one module manually

```bash
cd build
node module03.js
cp Module_03_*.pptx ../slides/
```

### Build all manually

```bash
cd build
node build-all.js
cp *.pptx ../slides/
```

> Always run scripts from inside `build/` — `require('./shared')` resolves relative to the script location.

## shared.js — The Three Shared Utilities

Every module script imports:

```js
const { C, shadow, icon } = require("./shared");
```

### C — Corporate Slate Palette

```js
const C = {
  navy:    "1C3557",   // primary dark background
  iceBlue: "5B8DB8",   // secondary text on dark
  pale:    "D4E4F0",   // body text on dark
  white:   "FFFFFF",
  offWhite:"F3F6F9",   // card/panel backgrounds on light slides
  accent:  "3A7DC9",   // primary accent blue
  teal:    "4A7FA8",   // secondary teal
  mid:     "2E5073",   // mid-dark panel fill
  text:    "1E2D3D",   // body text on light
  muted:   "7A90A8",   // secondary/caption text on light
  green:   "3A7E6E",   // success, positive
  steel:   "8096B0",   // neutral steel
  red:     "B03040",   // error, critical
  amber:   "B87820",   // warning, medium
};
```

Always use `C.navy`, `C.accent` etc. Never raw hex strings in module scripts.

### shadow() — Drop Shadow Factory

```js
// ✅ Always call fresh — never store and reuse
s.addShape(..., { shadow: shadow() });
s.addShape(..., { shadow: shadow() });
```

`pptxgenjs` mutates shadow objects in place. Reusing one object breaks all but the first shape.

### icon() — React Icon → Base64 PNG

```js
const ic = await icon(FaRobot, "#" + C.accent);
s.addImage({ data: ic, x: 0.5, y: 0.5, w: 0.4, h: 0.4 });
```

- Import icons from `react-icons/fa`
- Pass the component reference (not a string name)
- Color must include the `#` prefix
- Always `await` — it is async
- **Never call inside `.forEach()`** — use `for` loops (see Critical Rules)

## Critical Rules

### 1. Never await inside .forEach()

```js
// ❌ BREAKS silently
items.forEach(async (item) => {
  const ic = await icon(item.icon, "#" + item.color);
});

// ✅ CORRECT
for (let i = 0; i < items.length; i++) {
  const ic = await icon(items[i].icon, "#" + items[i].color);
}
```

### 2. Never reuse a shadow object

```js
// ❌ Second shape will be broken
const sh = shadow();
s.addShape(..., { shadow: sh });
s.addShape(..., { shadow: sh });

// ✅ Call fresh every time
s.addShape(..., { shadow: shadow() });
s.addShape(..., { shadow: shadow() });
```

### 3. Never use raw hex strings — always use C.*

```js
fill: { color: "#1C3557" }   // ❌
fill: { color: C.navy }      // ✅
```

### 4. Run scripts from inside build/

```bash
cd build && node module03.js   # ✅ shared.js resolves correctly
node build/module03.js          # ❌ require('./shared') breaks
```

## Common Tasks for Claude Code

### Fix text in a slide
```bash
grep -n "the text to find" build/module03.js
# edit the line
cd build && node module03.js
cp Module_03_*.pptx ../slides/
git add slides/Module_03_Context_Engineering.pptx
git commit -m "fix: correct text in module 03 slide 7"
```

### Add a slide to an existing module
1. Open `build/moduleNN.js`
2. Find the slide block to insert after (marked `// SLIDE N — Title`)
3. Add a new `{ const s = pres.addSlide(); ... }` block
4. Rebuild and copy to slides/

### Add a brand new module
1. `cp build/module11.js build/module13.js`
2. Update `pres.title`, all slide content, and filename in `writeFile()`
3. Add `"module13.js"` to the modules array in `build-all.js`
4. Update the module table below and in README.md

### Update content in a markdown guide
Edit `modules/NN-name.md` directly — these are plain text, no build step needed.
